# Код-стайлгайд

Last updated: 2026-02-16

## API Валидация

- Для `query`, `params` и `body` используем DTO + `class-validator` + глобальный `ValidationPipe`.
- Для enum-полей в запросах используем `@IsEnum(...)` в DTO.
- В контроллерах не пишем ручной парсинг/валидацию query-значений в виде приватных helper-методов, если это можно выразить через DTO.
- Type guard-функции выносим в отдельные файлы (`utils`/`guards`) только для сложных runtime-структур, которые не покрываются DTO-валидацией.

## Контроллеры

- Контроллер должен быть тонким: принять DTO и делегировать работу сервису.
- Бизнес-логика и преобразования данных живут в сервисах.

## Сервисы

- Публичные и сложные методы должны иметь короткие JSDoc-комментарии с назначением.
- Сервисы не должны дублировать валидацию, уже покрытую DTO/ValidationPipe, без необходимости.
